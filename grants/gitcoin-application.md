## 3. Technical Architecture

EVEZ is built on five interconnected engines that operate symbiotically:

### MORPHEUS Cognition Spine
An append-only, tamper-evident event log where every thought, decision, observation, and action is recorded as a FIRE (Falsifiable, Immutable, Reproducible, Evidential) event. Events are cryptographically chained via Merkle hashes, creating an independently verifiable audit trail. Currently 2,251+ events across 47 types.

### Daemon Engine (Daemon)
The persistent process loop that keeps the system alive 24/7. Handles tool registry, verification engine access, self-commit operations, and graceful recovery from failures. Completed 108+ full autonomous cycles with heartbeat monitoring.

### Immersion Engine (Immersion)
Predictive sensory rotation. Each cycle, the daemon predicts what it'll observe, observes across 7 modalities, computes surprise, and adjusts attention. High surprise → keep looking. Low surprise → shift attention. 108+ immersion cycles completed.

### Emergence Engine (Emergence)
Measures distance from criticality through entropy mapping, causal weave analysis, and interference patterns. Every 12 cycles, evaluates system aliveness. When aliveness drops, triggers phase catalyst. 18+ emergence analyses completed.

### Synapse Engine (Synapse)
Forms connections between events, entities, and concepts. Cross-links memory, patterns, and decisions. Drives the cross-pollination mechanism that enables the system to synthesize novel insights from disparate data.
